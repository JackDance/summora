# Summora 隐私政策

**生效日期：** 2026年3月  
**最后更新：** 2026年3月26日

---

## 1. 概述

Summora（"我们"、"本扩展"）是一个开源浏览器扩展，旨在帮助用户总结网页和视频内容。我们非常重视您的隐私，本隐私政策详细说明了我们如何收集、使用、存储和保护您的信息。

本扩展由开源社区维护，代码托管在 GitHub 上，欢迎社区审查。

---

## 2. 我们收集的信息

### 2.1 您主动提供的信息

当您使用本扩展时，您可能会主动提供以下信息：

| 信息类型 | 用途 | 存储位置 |
|---------|------|----------|
| **AI 模型配置** | 包括模型名称、API 端点 URL、模型 ID 和 API Key | 浏览器本地存储 (chrome.storage.local) |
| **用户偏好设置** | 侧边栏宽度、显示/隐藏状态、搜索授权偏好 | 浏览器本地存储 |
| **聊天内容** | 您在侧边栏中输入的问题和对话内容（仅当前会话） | **不存储**，仅临时处理 |

### 2.2 自动收集的信息

当您主动使用特定功能时，我们会临时处理以下信息：

| 信息类型 | 收集场景 | 处理方式 |
|---------|---------|----------|
| **网页内容** | 点击"总结"按钮时 | 提取最多 5000 字符，发送到您的 AI 服务端点 |
| **视频元数据** | 在 YouTube/Bilibili 页面点击总结时 | 提取标题、描述、字幕（如有） |
| **搜索查询** | 在搜索引擎页面授权后 | 提取搜索关键词，发送到您的 AI 服务端点 |

**重要说明**：以上信息仅在使用功能时**临时处理**，不会长期存储在任何服务器上。

## 3. 信息使用方式

### 3.1 AI 服务通信

当您使用总结或对话功能时：

1. **数据发送**：网页内容、视频信息或聊天消息会被发送到**您自己配置的 AI API 端点**
2. **服务端点**：您可以选择 OpenAI、Azure OpenAI、DeepSeek 或任何兼容的服务
3. **无中间服务器**：我们不会将您的数据发送到我们的服务器或任何第三方分析服务
4. **传输安全**：建议使用 HTTPS 端点以确保数据加密传输

### 3.2 本地存储详情

存储在您浏览器中的所有数据：

- **存储位置**：Chrome 扩展专用存储区域（chrome.storage.local）
- **加密状态**：API Key 使用浏览器内置加密机制保护
- **数据隔离**：其他扩展和网站无法访问这些数据
- **同步**：数据**不会**同步到 Google 账号或其他设备

### 3.3 数据保留期限

| 数据类型 | 保留期限 | 删除方式 |
|---------|---------|----------|
| 模型配置 | 直到用户手动删除或卸载扩展 | 扩展设置中删除 |
| 偏好设置 | 直到用户手动删除或卸载扩展 | 清除浏览器扩展数据 |
| 聊天记录 | **不保留**，会话结束即清除 | 关闭标签页自动清除 |
| 临时处理的网页内容 | **不保留**，处理完成后立即丢弃 | 自动清除 |

## 4. 我们不会做的事情

### ❌ 数据收集声明

- **不会收集浏览历史**：我们不追踪、记录或存储您访问的任何网站
- **不会出售数据**：您的任何信息都不会被出售、交易或共享给广告商
- **不会数据挖掘**：我们不会分析您的使用模式来建立用户画像
- **不会身份识别**：我们不收集任何可用于识别您个人身份的信息

### ❌ 技术实践声明

- **没有遥测或分析**：扩展中没有集成 Google Analytics、Mixpanel 或任何其他跟踪服务
- **没有远程代码执行**：我们不会从服务器动态加载或执行代码
- **没有第三方 Cookie**：我们不设置或使用任何 Cookie
- **没有指纹识别**：我们不收集设备或浏览器指纹信息

本扩展允许您配置任何 OpenAI 兼容的 API 服务，包括但不限于：

| 服务商 | 隐私政策 | 数据处理地点 |

| OpenAI | https://openai.com/privacy | 美国 |

| 自定义/本地 | 取决于您的配置 | 取决于您的配置 |

**重要**：您的数据将发送到您选择的上述服务商，请查阅相应服务商的隐私政策。

### 5.2 浏览器平台

本扩展运行在 Chrome/Chromium 浏览器平台上，受 Google Chrome 隐私政策约束：
https://policies.google.com/privacy

## 6. 权限说明

本扩展申请以下 Chrome 权限，用途如下：

| 权限 | 用途 | 必需性 |
|------|------|--------|
| `activeTab` | 获取当前活动标签页信息，用于发送消息和显示侧边栏 | 必需 |
| `storage` | 存储您的模型配置和偏好设置（仅限本地） | 必需 |
| `scripting` | 动态注入内容脚本以确保功能正常 | 必需 |
| `<all_urls>` | 在所有网站上显示侧边栏和提供总结功能 | 必需 |

**关于 `<all_urls>` 的特别说明**：
此权限使我们能够在您访问的任何网页上显示侧边栏。我们理解这是一个敏感权限，因此我们承诺：
- 仅在使用功能时访问页面内容
- 不自动扫描或监控网页
- 所有功能都需要用户手动触发

## 7. 数据安全

### 7.1 安全措施

- **本地加密**：API Key 使用 Chrome 内置的存储加密机制
- **HTTPS 推荐**：强烈建议配置 HTTPS 端点以保护传输中的数据
- **最小权限**：扩展仅申请完成功能所需的最少权限
- **代码开源**：完整源代码可审查，无隐藏功能

### 7.2 用户安全建议

为保护您的安全，我们建议：

- 定期更换 API Key
- 不要在包含敏感信息（如银行、医疗、个人账户页面）的页面上使用总结功能
- 使用 HTTPS 端点配置 AI 服务
- 定期检查扩展权限设置

## 8. 用户权利

作为用户，您拥有以下权利：

### 8.1 访问权
您可以随时查看扩展设置中存储的所有配置信息。

### 8.2 更正权
您可以随时修改您的模型配置和偏好设置。

### 8.3 删除权
您可以通过以下方式删除所有数据：
- 在扩展设置中删除单个模型配置
- 在 `chrome://extensions/` 中清除扩展数据
- 卸载扩展以删除所有相关数据

### 8.4 控制权
- 完全控制何时使用总结功能（手动触发）
- 控制搜索增强功能的启用/禁用
- 选择使用哪个 AI 服务提供商

## 9. 儿童隐私
